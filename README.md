# 📌 RetailPulse-Customer-Analytics-Relational-SQL-Pipeline
# 1. Project Overview
RetailPulse is a corporate-grade, end-to-end data analytics workflow designed to transform raw retail datasets into actionable business intelligence. This project demonstrates proficiency in the entire data pipeline: from initial cleaning and Exploratory Data Analysis (EDA) to complex SQL-based stakeholder reporting and professional-grade visualization.

This project demonstrates an end-to-end data analysis lifecycle, covering:
* **Data Cleaning & Wrangling** using Python (Pandas).
* **Advanced SQL Analysis** for business inquiry resolution.
* **Executive-level Reporting & Storytelling**.

## 2. Business Problem
RetailPulse faced challenges in understanding customer purchasing behavior. This analysis addresses six key business pillars [cite: 8, 10]:
1. **Low Customer Retention**: Identifying frequent vs. one-time buyers.
2. **Ineffective Marketing**: Assessing the ROI of discounts and promo codes.
3. **Product Demand**: Pinpointing high-performing categories and revenue generators.
4. **Payment Behavior**: Analyzing customer transaction preferences.
5. **Seasonal Variation**: Mapping sales trends.
6. **Customer Segmentation**: Profiling customers as new, returning, or loyal.

# 3. 🚀 Key Workflow
Data Preparation & Modeling: Data cleaning and structural organization to ensure relational integrity.

Exploratory Data Analysis (EDA): Performed deep-dive analysis using Python libraries (Matplotlib and Seaborn) to identify patterns, correlations, and outliers.

Data Analysis (SQL): Extracted strategic insights using advanced SQL techniques, including Window Functions (RANK(), PARTITION BY), complex CASE statements, and subqueries.

Business Reporting: Synthesized technical findings into high-impact strategic recommendations for stakeholders.

# 4. 📊 Strategic Insights Derived
Revenue Optimization: Identified that Electronics is the primary revenue driver (480,210.81).

Discount Effectiveness: While discounts don't drastically alter total revenue, they increase the average purchase value by ~20.8%, serving as a potent lever for per-transaction growth.

Customer Segmentation: The base is heavily skewed toward Loyal Customers (3,116). We identified an opportunity to improve subscription rates among "Repeat Buyers," who currently show a lower subscription propensity (29.64%) than "Normal Buyers" (37.25%).

Demographic Targeting: The 51+ age demographic is the company's highest revenue contributor (399,214.36), suggesting a need for tailored marketing campaigns for this group.

Operational Efficiency: Faster shipping methods (specifically Express) correlate with a significantly higher average order value (354.72).

# 5.  🛠 Tech Stack
Data Manipulation: Python (Pandas)

Visualization: Matplotlib, Seaborn

Database: MySQL (Relational Modeling)

Analytical Techniques: Statistical trend analysis, churn behavior profiling, and categorical performance benchmarking.
